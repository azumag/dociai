# Architecture

## 方針

最初のPoCはローカルHTMLアプリとして作ります。将来的には、ローカル常駐アプリ、OBSプラグイン、Webサービス版に拡張できるように、ブラウザ固有処理とコア処理を分離します。

## 処理フロー

1. コメント、手動入力、ニュース、話題、ショートカットなどのイベントを受け取る
2. `TriggerEngine` が反応すべきイベントか判定する
3. `ContextBuilder` がコメント履歴、配信画面の状態、ニュース/話題文脈をまとめる
4. `PersonaRouter` がペルソナとAIコネクタを選ぶ
5. `AIConnector` がモデルへ問い合わせる
6. `SpeechQueue` が音声読み上げを順番に実行する
7. ログにコメント、選択ペルソナ、応答、読み上げ状態を残す

## 文脈管理

コメント履歴は長時間配信で肥大化するため、3層に分けます。

| 種別 | 内容 |
|---|---|
| 生ログ | 直近80件程度のコメント |
| プロンプト投入 | 直近10から20件 |
| 長期文脈 | 数分ごとに要約した `streamSummary` |

画面文脈は、毎コメントで画像認識せず、手動更新、ショートカット更新、一定時間ごと、特定キーワード時に限定します。

## トリガー

初期PoCでは以下を実装候補にします。

| トリガー | 内容 |
|---|---|
| `keyword` | 特定ワードを含むコメントで発火 |
| `hotkey` | ショートカットキーで発火 |
| `interval` | 一定時間ごとに発火 |
| `manual` | UIボタンで発火 |
| `random` | コメントに一定確率で反応 |

## セキュリティ

APIキーは原則として永続保存しません。

- `config.local.json` はGit管理しない
- LocalStorageにはAPIキーを入れない
- ファイル選択で読み込んだキーはメモリ保持のみ
- 将来的にはローカル常駐アプリでOSのKeychain/Credential Managerに保存する
- 公開サービス版ではサーバー側で暗号化保存する
